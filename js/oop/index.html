Is there any difference between these two class definitions
In the first class definition, when the class is created, a new 'say_hi' is created just for that class. --------- In the second class definition, just one 'say_hi' function is created and is used for all MyClass2's since MyClass2.prototype is the parent of MyClass2. When the MyClass2 object is created, it inherits the properties (say_hi) of its constructor's prototype. second method is more efficient.
Add methods getter/setter methods (i.g. get_name(name), set_name()) to the classes and change say_hi() text to be "Hi, my name is %name%"!
see code below: // Add getters & setters function MyClass1 () { this.say_hi = function() { console.log('Hi, my name is ' + name + '!'); }; // name var name = name; this.get_name = function() { return name; } this.set_name = function(value) { name = value; } } function MyClass2 () { // name this.get_name = function() { return name; } this.set_name = function(value) { name = value; } } MyClass2.prototype.say_hi = function() { console.log('Hi, my name is ' + name + '!'); }; var inst1 = new MyClass1(), inst2 = new MyClass2(); inst1.set_name('Class1'); inst2.set_name('Class2'); inst1.say_hi(); inst2.say_hi();
Change class definition to implement the Singleton pattern
see code below. I have not had to implement a singleton pattern before (or the interaction of singleton and javascript's prototype), so I researched as much as I could on how to implement this. var inst1 = new function() { this.name = "Class1"; this.say_hi = function() { console.log('Hi, my name is ' + this.name + '!'); }; } var MyClass2 = function() { this.name = "Class2"; } MyClass2.prototype.say_hi = function () { console.log('Hi, my name is ' + this.name + '!'); }; var inst2 = new MyClass2(); inst1.say_hi(); inst2.say_hi();
